Fragola is an event-driven SDK for building agents without hiding the underlying OpenAI message model.

The core primitives stay small:

- **Agents** own instructions, messages, tools, and runtime state.
- **Tools** are typed functions the model can call.
- **Events** expose the model and tool lifecycle.
- **Hooks** package reusable behavior and integrations.
- **Stores** hold shared mutable runtime data.
- **Metadata** lets you keep typed app data attached to messages.

## Start here

- Read [Quickstart](/get-started/quickstart) for the shortest path from install to first reply.
- Read [Agent with tools](/get-started/agent-with-tools) to see the main production flow.
- Read [Work with metadata](/agents/work-with-metadata) if you need request IDs, trace data, or cache hints on messages.
- Browse [Events](/events) when you want to intercept model calls, tool execution, or state updates.

## What is ready

- Agent creation and message loops.
- Tool execution with Zod or JSON-schema-style definitions.
- Event hooks around model invocation, tools, and state updates.
- Typed metadata that stays in `state.messages` and is stripped only before the outbound API request.

## What comes next

- External hook packages live in this repository under `hook.presets/`, including guardrails, orchestration, and MCP tools.

## Installation

```bash
npm install @fragola-ai/agent
```

```typescript
import { Fragola } from "@fragola-ai/agent";

const fragola = new Fragola({
  apiKey: process.env.OPENAI_API_KEY!,
  model: "gpt-4o-mini"
});

const agent = fragola.agent({
  name: "docs-assistant",
  description: "A simple starting point",
  instructions: "You are a friendly assistant."
});

const state = await agent.userMessage({
  content: "Give me three names for a docs rewrite project."
});

console.log(state.messages.at(-1));
```
